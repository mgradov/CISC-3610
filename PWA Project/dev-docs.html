How to change the dataset

The app reads everything from data.json. To swap in your own dataset (for example, planets or musical instruments), edit that file.

The structure looks like this:

{
  "animals": [
    {
      "title": "African Elephant",
      "habitat": "Savannas of Africa",
      "description": "The African elephant is...",
      "image": "https://example.com/elephant.jpg",
      "audio": ""
    },
    ...
  ]
}

Steps

  1. Open data.json in a text editor.
  2. Keep the outer object and the "animals" array.
  3. Replace each entry, or add new ones, using the same fields: title, habitat, description, image, and audio.
  4. Make sure the JSON is valid: commas between entries, no trailing comma after the last one, double quotes around all keys and string values.
  5. Save the file and refresh the app.

Field meanings

  • title — shown on the menu button and as the heading
  • habitat — shown after the bold "Habitat:" label
  • description — the main paragraph, also what the Listen button reads aloud when audio is empty
  • image — URL of the image (remote URL or local file path)
  • audio — URL of an audio file to play; if empty, the app uses the browser's built-in text-to-speech on description

Using a different topic

If your dataset is about a different topic, you can rename the outer key from "animals" to something else (like "planets"). If you do that, also change this line in app.js:

animals = data.animals;

to match the new key, for example:

animals = data.planets;

You can also rename the habitat field to fit your topic (such as "location" or "era"); if you do, update the corresponding line in app.js inside showAnimal().

Adding more files to the offline cache

If you add new files (such as local images or audio files), list them in the FILES_TO_CACHE array at the top of service-worker.js, and bump the CACHE_NAME version string (for example from wild-animals-v1 to wild-animals-v2) so returning users get the update.
